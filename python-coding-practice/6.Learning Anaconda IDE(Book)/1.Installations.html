1.1. Installing Python 3 on macOS¶

There are several ways to install Python on Mac. The first option is using Homebrew, which is unofficial package manager of macOS. Homebrew brings many advantages along with it. So, having it makes life easier for macOS users.

First, check if Homebrew is installed:

brew

If we have an output as “-bash: brew: command not found,” it unfortunately means that Homebrew is not installed. Otherwise, it would be ready to use. Just make sure that we are using the newest version and run the following command:

brew update

For those who do not have Homebrew, we need to have XCode Command Line Tools so that we can smoothly install the Homebrew and Python. If we do not have XCode Command Line Tools, please go to the XCode website and download and install XCode.

Lastly, we can install XCode Command Line Tools by running the following command in our terminal:

xcode-select --install

By installing the XCode Command Line Tools, we are ready to install Homebrew. Now, please paste the below given one line code in the folder named brew.sh and click "Enter":

/usr/bin/ruby -e "$(curl -fsSL \ https://raw.githubusercontent.com/Homebrew/install/master/install)"

Now, confirm that if the installation work. The following command does it for us:

brew update

If everything is up and running, we are ready to move forward and install Python3 via this command:

brew install python3

This command installs Python3 and allows interactive Python interpreter to run the python3 command.

It is time to confirm that installation is done in a perfect way:

python3 --version

If we properly install the Python, this command will return the latest version of Python. If so, congrats!!!

1.3.1. Cells¶

Jupter Notebook works via cells, which contains everything we put in the Notebook. Right now, we are working in a cell. Nothing happens when we type something inside the cell, we need to run it, and in order to run it we need to click on the run button at the top of the Notebook. Alternatively, after selecting the cell ctrl + Enter can be used, too.

A notebook can have as many cells as we need. There are several ways to add cells. When we run the last cell in the notebook, a new cell is automatically added. Also, if we have a selected but inactive cell, whose color is blue, we can press a if we want to add a cell above that cell, or b if we want to add a cell below it. To delete a cell, press x.

Each notebook runs in a Python environment. Codes are run in an orderly way. That is to say, the output of the preceding code will be the input of the subsequent one. If we do not pay attention to the ordering of the codes in the cells, we may encounter some coding issues. If we change the code in our cell or run it multiple times, we may have changed the variables or output in an undesired way. The same thing can also happen if we run cells in the wrong order. That's why it is very important that all our code is sequential and executable!

We can start typing our code inside the cell, which is surronded by green frame. As default, Jupyter notebook uses Code, which allows we to type our code in the cell.

Please note that the drop down menu indicated by the red arrow allows us to type either the code or write down text, along with other options such as a heading. If we click on this drop down menu, we are going to see 4 options, which are:

  • Code
  • Markdown
  • Raw NB Convert
  • Heading

1.3.2. Markdown¶

So far we have only talked about code cells, but there are other types of cells. One of the important cell types we need to know is the markdown cell. This cell is actually a marking cell. We can double-click the markdown cell and edit it. We can see what the output looks like by running the cell.

When we want to use a Jupyter Notebook to write a report, we may want to write notes that would not be convenient using comments in the Python code (comments that begin with the # sign and do not affect the code). Markdown cells can be used for this purpose. By default, each new cell is a code cell. To convert a cell into a markdown cell, select the cell, make sure it is inactive and press m. It's that simple. Or, as shown before, go to drop down menu and select Markdown.

2.4.3. Histogram¶

A histogram is a great tool to show the possible values of a variable and how frequent these values are. In addition to that, histogram gives us an initial idea about the distribution of the data has. It may be quite important to know the distribution of a data before running a model because some models assume a certain type of distribution.

The histogram begins by dividing the values into bins. These bins are equal-sized ranges with an upper and lower limit. The histogram then counts how many values are in each bin and draws a bar corresponding to this number for each bin. This gives us an idea of the frequency of the range of its variable. So, histograms are great for visualizing the distribution of a variable.

Here, we draw the histogram of the total_profit and total_units variables.

2.4.4 Box Plot¶

Another useful visualization is the boxplot. We can create a boxplot with the .boxplot() function. A boxplot is a relatively simple visualization. Let's create one and then examine its properties.

In [100]:
plt.figure(figsize=(10, 5))

plt.subplot(1, 2, 1)
plt.boxplot(df.facewash)
plt.ylabel('Face Wash')

plt.subplot(1, 2, 2)
plt.boxplot(df.bathingsoap)
plt.ylabel('Bathing Soap')

plt.tight_layout()
plt.show()

The boxplot consists of a box containing what is called quartile (interquartile range or IQR for short). First of all, let's explain what the quartile means. We can divide a variable into four quartiles based on the values it contains:

  • From the lowest value to 25%
  • From 25% to the median
  • From the median to 75%
  • From 75% to the maximum value

These groups are called quartiles and the values that the two in the middle quartiles cover are called interquartile range. Thus, the box we see in the boxplot contains all the data from 25% to 75% and shows the median with a red line.

There are vertical lines called whiskers above and below the box (for this reason, the boxplots are sometimes called "box and whisker diagrams"). The whiskers extends up and down 1.5x from the top and bottom of the IQR box. If all values are in this range, the whiskers goes up to the maximum and minimum values of the variable. Any value beyond 1.5 times the IQR represents as a tick beyond the end of the whiskers. For example, in the line above on the left hand side, there are some values above the whiskers.

Boxplots are not useful for learning about the distribution of a variable. However, it is very useful for eyeballing the range of values and for detecting possible outliers. Data points marked with a dot or circle on a box diagram are generally considered to be outliers. Therefore, boxplots are widely used when working on new data. For more information about boxplots, see the .boxplot() documentation.
